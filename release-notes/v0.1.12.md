# Release v0.1.12 (draft)

Release highlights

- Patch release: bumped package version to **v0.1.12**. ✅
- Dependency and security maintenance: upgraded npm packages and resolved audit findings to zero vulnerabilities. 🔒
- ESLint 10 compatibility: migrated from `eslint-plugin-import` to `eslint-plugin-import-x` and updated related rules. 🛠️

## Details

### Dependency and security updates 🔒

- Updated direct npm dependencies/devDependencies to newer versions.
- Regenerated lockfile and applied override updates so `npm audit` reports no vulnerabilities.

### Linting updates 🛠️

- Replaced `eslint-plugin-import` with `eslint-plugin-import-x`.
- Updated lint rules from `import/*` to `import-x/*`.
- Removed `eslint-plugin-promise` rules and retained promise safety guardrails through existing core/TypeScript ESLint rules.

### Validation status for this draft

- `npm run lint`: passes (warnings only, no errors).
- `npm test`: fails in `src/common/pipes/image-validation.pipe.spec.ts` because the runtime environment is missing the OS `file` binary required by `mime-detect`.
- `npm run test:e2e`: passes.
- `npm run test:prod`: did not complete within the execution timeout in this environment (captured as a blocker for final release sign-off).

## Upgrade notes

- No intentional API breaking changes in this release.
- Before publishing this draft release, re-run `npm test` and `npm run test:prod` in an environment where required OS tooling (including `file`) is available.
