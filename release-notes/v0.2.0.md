# Release v0.2.0

Release highlights

- **Major infrastructure migration**: the project moved from CommonJS / Jest to native ESM + Vitest. 🏗️
- TypeScript now emits pure ESM output (`"module": "NodeNext"`, `"target": "ES2024"`); `package.json` declares `"type": "module"`. All relative imports carry explicit `.js` extensions per Node's ESM resolver rules. ⚡
- Jest packages and all Jest configuration files have been removed. The test runner is now Vitest v4 with a single workspace config covering unit, E2E, live-E2E, and production test suites. 🧪
- ESLint updated: `eslint-plugin-jest` replaced with Vitest global recognition; CommonJS-specific overrides removed. 🛠️
- CI/CD updated to Node 24 and Vitest commands, with JUnit reports at `./junit/vitest-junit.xml` and `lcov` coverage for SonarQube. 🔄
- Dead npm scripts (`verify:assessor`, `dev:delegate`) and unused dependencies (`ts-node`, `tsconfig-paths`) removed. 🧹
- Project documentation (`docs/`, `README.md`) systematically updated from Jest to Vitest. 📚

## Details

### ESM migration 🏗️

- **TypeScript configuration** — `tsconfig.json` now uses `"module": "NodeNext"`, `"moduleResolution": "NodeNext"`, and `"target": "ES2024"`. The `types` array dropped `"jest"` (now only `["node"]`). The standalone `tsconfig.test.json` (a Jest ESM workaround) has been deleted.
- **Package declaration** — `package.json` gained `"type": "module"`, marking all `.js` files as ESM by default. The production build (`nest build`) now emits pure ESM output via `tsc`.
- **Entry-point detection** — `src/main.ts` and `src/testing-main.ts` use an ESM-compatible `isRunningDirectly()` based on `import.meta.url` and `pathToFileURL`, replacing the old CommonJS `require.main === module` pattern and the `JEST_WORKER_ID` guard.
- **Import extensions** — All relative imports across source, spec, and test files now carry explicit `.js` extensions (`from './foo'` → `from './foo.js'`), as required by Node's ESM resolver under `NodeNext` module resolution. This affected approximately 160 imports across 50+ files.
- **JSON import attribute** — `status.service.ts` (and its spec) now use the `with { type: 'json' }` import attribute for `package.json`, and access the version via `packageJson.default.version` to align with Node ESM semantics.
- **Path-alias imports removed** — `src/v1/assessor/assessor.controller.ts` no longer uses `src/...` path aliases; these were converted to relative imports to avoid requiring a runtime path-alias loader.
- **Health-check script** — `scripts/health-check.js` converted from CommonJS to ESM (`import http from 'node:http'`).

### Test runner migration 🧪

- **Jest removed, Vitest v4 installed** — All Jest dependencies (`jest`, `@types/jest`, `@jest/globals`, `jest-junit`, `ts-jest`, `eslint-plugin-jest`) and all Jest configuration files (9 files including `jest.config.js`, four `jest-e2e.*.config.cjs` files, `jest.setup.ts`, and two E2E setup files) have been deleted.
- **Vitest workspace** — A single `vitest.config.ts` (using Vitest v4's `test.projects` API) manages four projects: `unit` (source specs), `e2e` (mocked E2E), `e2e-live` (live Gemini E2E), and `prod` (Docker image tests). Global setup in `vitest.setup.ts` handles `reflect-metadata` import ordering, environment variables, and `dotenv` loading.
- **E2E LLM mocking** — The old CommonJS `--require` shim (`test/utils/llm-http-shim.cjs`) has been replaced by an ESM `--import` preload shim (`test/utils/llm-mock.mjs`). This patches `GoogleGenerativeAI.prototype.getGenerativeModel` in the spawned child process before application code runs. `vitest.e2e.setup.ts` sets `process.env.E2E_MOCK_LLM = 'true'` to activate the mock. The `app-lifecycle.ts` module was updated to use `--import=<file://...>` instead of `--require`.
- **Serial E2E execution** — The E2E, live-E2E, and prod projects have `fileParallelism: false` to prevent port collision (`EADDRINUSE`) when spawning multiple app instances. Unit tests remain parallel for speed.
- **JUnit reports** — Vitest's built-in JUnit reporter outputs to `./junit/vitest-junit.xml` (replacing the old `jest-junit` package path `./junit/jest-junit.xml`).
- **Coverage** — Vitest with `@vitest/coverage-v8` provides coverage via `npm run test:cov`, producing `text`, `html`, `lcov`, and `clover` output. The `lcov` report at `coverage/lcov.info` feeds SonarQube analysis.

### ESLint configuration updates 🛠️

- `eslint-plugin-jest` removed along with its plugin registration, recommended rules, and `globals.jest`.
- Vitest globals (`vi`, `describe`, `it`, `expect`, `beforeEach`, etc.) recognised via `globals.vitest` from the `globals` npm package. The `eslint-plugin-vitest` package was installed but found incompatible with ESLint 10.x and subsequently uninstalled; the `globals` approach is sufficient.
- CommonJS-specific overrides removed: `**/*.cjs` ignore block dropped; `unicorn/prefer-module` and `unicorn/prefer-top-level-await` disabled overrides for `main.ts`/`testing-main.ts` no longer needed (both files now use ESM-native patterns).
- The `unicorn/prefer-uint8array-base64` override for `image-validation.pipe.ts` is retained (deferred — migrating `Buffer.from(…, 'base64')` to `Uint8Array.fromBase64()` is a separate code change scoped outside this migration).
- `coverage/` added to ESLint's top-level ignores so generated coverage artifacts are not linted.

### CI/CD updates 🔄

- **GitHub Actions** (`.github/workflows/ci.yml`) updated across all jobs:
  - Node version bumped from 22 to 24 (matching the Dockerfile's `node:24-alpine`).
  - Unit job: `npm test -- --verbose --coverage` replaced by `npm run test:cov`.
  - E2E job: `npm run test:e2e -- --verbose` replaced by `npm run test:e2e`.
  - JUnit report paths updated from `./junit/jest-junit.xml` to `./junit/vitest-junit.xml`.
- **npm scripts** — All test scripts target Vitest: `test` (unit), `test:watch`, `test:cov`, `test:debug`, `test:e2e` (alias for `test:e2e:mocked`), `test:e2e:mocked`, `test:e2e:live`, and `test:prod`. Each uses `vitest run --project <name>`.

### Dependency and script cleanup 🧹

- **Jest ecosystem removed** — `jest`, `@types/jest`, `@jest/globals`, `jest-junit`, `ts-jest`, and `eslint-plugin-jest` uninstalled.
- **Unused tooling removed** — `ts-node` and `tsconfig-paths` uninstalled (no remaining scripts depend on them).
- **Dead npm scripts removed** — `verify:assessor` and `dev:delegate` scripts removed from `package.json`; their target script files (`scripts/verify-assessor.ts`, `scripts/codex-delegate.ts`) had been deleted in earlier cleanup commits.
- **SonarQube alignment** — The `sonar-project.properties` comment updated from `# Jest/SonarCloud` to `# Vitest/SonarCloud` (coverage path `coverage/lcov.info` already matched).

### Documentation updates 📚

- **13 project documentation files** updated systematically from Jest to Vitest references, covering:
  - `README.md` — tech stack and testing commands.
  - `docs/architecture/overview.md` — testing framework reference.
  - `docs/deployment/cicd.md` — coverage and JUnit report paths.
  - `docs/development/code-style.md`, `docs/development/debugging.md`, `docs/development/git-workflow.md`, `docs/development/workflow.md` — code examples and workflow references.
  - `docs/modules/utilities.md` — `getCurrentDirname()` Jest compatibility note.
  - `docs/testing/README.md`, `docs/testing/E2E_GUIDE.md`, `docs/testing/PRACTICAL_GUIDE.md`, `docs/testing/PROD_TESTS_GUIDE.md` — config file names, mocking guidance, and command examples.
- All changes use Vitest API (`vi.fn()`, `vi.mock()`, `Mock` from `vitest`, `vitest.config.ts`, `vitest run --project ...`, `llm-mock.mjs` + `--import` shim).
- `AGENTS.md` and `.opencode/agents/*.md` still contain Jest references; these are owned by other agent definitions and were intentionally left for their maintainers to update.

### Known limitations and notes

- **`npm run test:e2e:live`** requires a live Gemini API key (`GEMINI_API_KEY` in `.env`) and `data/tableTask.json` fixture data — these are not available in CI or default sandbox environments. The command is validated to parse and invoke Vitest correctly but cannot complete without credentials.
- **`npm run test:prod`** requires Docker to build and run the production image; it does not execute in CI sandboxes without Docker.
- **`AGENTS.md` and agent definition files** (`.opencode/agents/*.md`) still reference Jest in places. These are maintained by separate agent workflows and were not included in this migration's scope.
- **`unicorn/prefer-uint8array-base64` override** for `image-validation.pipe.ts` was retained. Migrating the 7 related `Buffer.from(…, 'base64')` calls to `Uint8Array.fromBase64()` is deferred to a future code-change release.
- **`eslint-plugin-vitest`** is incompatible with ESLint 10.x; Vitest global recognition is handled via the `globals` package instead.

## Upgrade notes

- **Breaking change**: the project no longer supports CommonJS module output. Any tooling or scripts that rely on CJS interop (e.g., `require()`-based test runners, `--require` preload hooks) will need updating.
- **Test runner**: if you run tests locally, ensure your Node.js version is 24 or later. Use `npm run test` for unit/integration tests, `npm run test:e2e` for mocked E2E tests.
- **CI pipelines**: if you maintain custom CI workflows, update them to use Node 24, Vitest commands (`npm run test:cov`, `npm run test:e2e`), and the JUnit report path `./junit/vitest-junit.xml`.
- **ESLint**: if you extend or customise the shared ESLint config, note that `eslint-plugin-jest` rules are no longer available; Vitest globals are provided via the `globals` package.
- **Dependency changes**: Jest packages and `ts-node`/`tsconfig-paths` have been removed. Run `npm ci` after pulling to ensure a clean dependency tree.
- **Import extensions**: all relative imports now require explicit `.js` extensions. If you add new files, ensure `from './foo.js'` syntax (even for `.ts` sources).
- **Documentation**: project documentation has been updated to reflect Vitest and ESM conventions. Review `docs/testing/` for current guidance.
