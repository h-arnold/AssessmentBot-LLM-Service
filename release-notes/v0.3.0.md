# Release v0.3.0 (draft)

Release highlights

- **Mistral LLM provider added.** Assessments can now be routed to Mistral
  models (`mistral-*`) alongside the existing Gemini provider, selected by model
  id at send time. 🚀
- **Model-name-based routing.** `RoutingLLMService` resolves the provider from
  the model id prefix and is now the application's `LLM_SERVICE_TOKEN`
  implementation. Gemini and Mistral services are registered as routing
  targets. 🔀
- **New environment variables.** `MISTRAL_API_KEY` (required for Mistral
  models), `DEFAULT_TEXT_TABLE_MODEL`, `DEFAULT_IMAGE_MODEL`,
  `TEXT_REASONING_EFFORT`, and `IMAGE_REASONING_EFFORT` configure the default
  provider/model and reasoning effort. Gemini and Mistral keys are
  independently required — neither is a fallback for the other. 🔑
- **Shared LLM error-mapper.** Provider error mapping has been consolidated
  into a single `classifyLlmError()` helper (`src/llm/llm-error-mapper.ts`).
  Both `GeminiService` and `MistralService` delegate to it with per-provider
  probe configuration; the private status-code/rate-limit/resource-exhausted
  helpers previously duplicated on `GeminiService` have been removed. 🧹
- **Reasoning-effort support.** The abstract `reasoningEffort` payload field is
  honoured by both providers (`off`/`low`/`high`/`max`), mapped to each SDK's
  native representation. 🧠

## Details

### Mistral provider 🚀

- `MistralService` (`src/llm/mistral.service.ts`) sends text and image
  assessments via the `@mistralai/mistralai` SDK and validates responses with
  the shared `LlmResponseSchema`.
- The existing E2E mock shim (`test/utils/llm-mock.mjs`) now patches
  `Mistral.prototype.chat` via a configurable prototype getter override, in
  addition to the Gemini mock, so the mocked E2E suite covers both providers.

### Routing architecture 🔀

- `DEFAULT_TEXT_TABLE_MODEL` / `DEFAULT_IMAGE_MODEL` now default to
  `mistral-small-latest`. The provider is resolved at send time from the model
  id prefix, so the Gemini `assessor.e2e-spec.ts` and `assessor-live.e2e-spec.ts`
  suites are explicitly pinned to Gemini models via `environmentOverrides` to
  preserve the Gemini regression coverage.

### Shared error mapper 🧹

- `src/llm/llm-error-mapper.ts` exports `classifyLlmError`, the
  `LlmErrorMapperProbes` interface, and `normaliseStatusCode`. Classification
  priority, retryability, and 4xx/5xx mapping are now defined once and tested
  with synthetic probes in `llm-error-mapper.spec.ts`.
- See `docs/llm/error-handling.md` for the updated "Worked Example" (now the
  shared helper) and a new "Mistral Provider" subsection.

### Environment configuration 🔑

- `docs/configuration/environment.md` documents all five new variables with
  their Zod types, defaults, and accepted enum values.

## Upgrade notes

- **New required key:** deployments that route to Mistral models must supply
  `MISTRAL_API_KEY`. Because the model defaults now point at `mistral-*`, a
  fresh deployment without `MISTRAL_API_KEY` will fail at `MistralService`
  construction. To stay on Gemini only, set `DEFAULT_TEXT_TABLE_MODEL` and
  `DEFAULT_IMAGE_MODEL` to `gemini-2.5-flash-lite` / `gemini-2.5-flash` and
  omit the Mistral key.
- No HTTP API contract changes for `/v1/assessor` consumers.
